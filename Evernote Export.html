Основные положения:
0.1) Помимо базовой аутентификации веб-сервиса (логин/пароль для авторизации) в каждый метод сервиса необходимо передавать токен, полученный по опред. алгоритму
0.2) Срок жизни сгенерированного токена составляет 600 секунд
0.3) Для генерации используются секретный код - SecretCode
0.4) Параметр "Token" всегда (!) идет последним в строке параметров, передаваемых в теле запроса http и является обязательным параметром во всех методах кроме метода "GetServerTime"
0.5) Для получения текущей даты/времени сервера используется метод GetServerTime() , который возвращает дату строкой в формате "yyyyMMddHHmmss"
0.6) Алгоритм генерации токена следующий:

Разд = "##"; 
СтрВремя = GetServerTime(); // используется значение "ServerTime" из ответа этого метода, строка в формате "yyyyMMddHHmmss"
СтрокаПараметров - строка параметров метода сервиса вида "Param1=Value1&Param2=Value2..."
СтрокаДанных = SecretCode + Разд + СтрВремя + Разд + СтрокаПараметров;
СтрокаДанных64 = Base64Строка(СтрокаДанных); // Строка данных преобразованная  в формат Base64
СтрокаТокен = ХэшSHA256(СтрокаДанных64);   // Хэш строки данных закодированной в Base64 полученный по алгоритму SHA256

По мимо параметров, передаваемых в конкретные методы, при каждом запросе передаются также мета-параметры
"OSVer" - строка, версия ОС клиента.
"OSVerNum" - строка версия ОС
"IsAndroid" - булево, запрос отправлен с приложения моб. приложения?;
"AppVer" - строка, версия приложения. (для первой версии SKOmed нумерация была вида 1.X.YY, для второй версии 2.ХХ.YY) , 
где XX - номер редакции (меняется при масштабных изменениях приложения), YY - минорный номер версии, меняется при публикации новой версии.

1.1 Метод "GetPatientByIIN"
Описание метода: 
С помощью этого метода можно получить данные о прикреплении пациента к поликлинике и его участковом враче.
Входные параметры: 
"IIN" - строка, ИИН пациента
"OrgID" - строка, ИД организации (не обязательный параметр)
Выходные параметры:  
"FIO" - строка, полное ФИО пациента
"Territory" - строка, наименование участка прикрепления
"TerritoryID" - строка, идентификатор участка прикрепления
"Attachment" - строка, наименование организации прикрепления
"AttachmentID" - строка, идентификатор организации прикрепления
"AttachmentAddress" - строка, адрес организации прикрепления
"Doctor" - строка, полное ФИО участкового врача пациента
"DoctorID" - строка, идентификатор участкового врача
"Address" - строка,, адрес места жительства пациента
"PhoneNumber" - строка,, контактный телефон пациента
"ServerDate" - строка,, текущая дата на сервере в формате "yyyyMMddHHmmss"
"ErrorDesc" - строка,, текстовое описание ошибки
"RegAvailable" - число
0 - запись в МО прикрепления пациента не доступна
1 - запись в МО прикрепления пациента доступна
"HomeCallAvailable"- число
0 - НЕ доступен вызов на дом участкового врача
1 - доступен вызов на дом участкового врача
"OrgErrors"[] - массив, структур с данными ошибок по всем организациям, для которых доступна запись
    --  "OrgID" - ИД организации 
    --  "Name" - Наименование организации;
    --  "ErrorText" - текст ошибки по организации. Важно!! - если это поле заполнено, запись в эту организацию не возможна. В этом случае не нужно показывать пользователю следующий шаг с выбором врача и временем записи.

"RegToProfileSpecs" - булево, доступна ли запись к узким специалистам?
"ShowRegularExamsNotice" - булево, выводить сообщение о осмотрах (глобально для организации)?

"ED_TextFluoro" - строка, текст наличия флюорографии.
"ED_TextExam" - строка, текст наличия осмотра.
"ED_TextRW" - строка, текст наличия анализа RW.
"ED_LastFluoroDate" - строка, дата последней флюорографии
"ED_LastExamDate" - строка, дата последнего осмотра
"ED_LastRWDate" - Строка, дата последнего анализа RW
"ED_FluoroIsNeeding" - булево, нужна ли флюорография?
"ED_ExamIsNeeding" - булево, нужен ли осмотр?
"ED_RWIsNeeding"- булево, нужен анализ RW?
"ED_ShowMessage"- булево, Истина, когда хотя бы один из параметров осмотров (флюорография, RW или см. кабинет) устарел
"ED_RegAvailable" - булево, доступна ли запись, (Ложь, если хотя бы одно из трех условий выше равно Истина)

"ErrorCode" - число, код результата выполнения метода:
    // 0      - успешное выполнение метода.
    // 10    - база МО не доступна!
    // 100  - ошибка при формировании данных пациента (означает, что что-то пошло не так при заполнении структуры данных).
    // 110  - ошибка проверки доступности записи в МО (разрешена ли запись пациента в МО прикрепления?).
    // 120  - ошибка, не найдена МО прикрепления пациента.
    // 130  - ошибка, не найден участок прикрепления пациента.
    // 140  - ошибка, не найден участковый врач пациента.
    // 200  - ошибка, ИИН пациента не заполнен.
    // 300  - ошибка получения данных пациента из МБ.

1.2 Метод "GetShedule"
Описание метода: 
С помощью этого метода можно получить список дней для записи пациента к определенному врачу.
Метод возвращает список дат, доступных для записи к врачу за разрешенный период записи (в наст. время - 7 рабочих дней).
Доступной датой считается дата, в которой есть хотя бы одно свободное время для записи.
Входные параметры (обязательные): 
"OrgID" - строка, ИД организации (значение поля "AttachmentID" из метода "GetPatientByIIN")
"DoctorID" - строка, идентификатор участкового врача (может быть пустым для организаций, у которых включена опция "DisableDoctorSelection" в результате метода "GetOrgListForAppointment")

"IIN" - ИИН пациента, используется для тестового пациента (см. описание демо ИИН в начале документации).

Выходные параметры:  
"Doctor" - строка, полное ФИО участкового врача пациента
"Dates"[] - массив структур с описанием не занятых дат для записи на прием к врачу

1.3 Метод "SaveAppointment"
Описание метода: 
С помощью этого метода можно создать запись пациента на прием к участковому врачу.
При записи проверяются следующие моменты:
  1. Пациент может быть записан к одному и тому же врачу только один раз в день
  2. Пациент не может быть записан к разным специалистам на одно и то же время или при пересечение времени визитов.
  3. Указанное время визита еще не занято. 
Входные параметры (обязательные): 
"IIN" - строка, ИИН записываемого пациента.
"OrgID" - строка, ИД организации (значение поля "AttachmentID" из метода "GetPatientByIIN")
"DoctorID" - строка, идентификатор участкового врача
"Date" - строка, дата приема в формате "yyyyMMdd"
"TimeStart" - строка, время начала приема в формате "HH:mm"
"TimeEnd" - строка, время окончания приема в формате "HH:mm"
"RecordingMethod" - число, значения 
1 - запись через мобильное приложение,
2 - запись через сайт,
3 - запись через терминал,
необязательные параметры:
"CabinetID" - строка, идентификатор кабинета, в котором врач ведет прием
"Reason" - строка, причина записи
"Language" - число, флаг языка в сообщениях от сервера 
1 - русский,
2 - казахский
Выходные параметры:  
"ReceiptNumber" - строка,, номер талона на прием.
"GUID" - строка,, уникальный идентификатор документа записи.
"RegDateTime" - строка,, дата в формате "yyyyMMddHHmmss", регистрации записи.
"ErrorDesc" - строка,, текстовое описание ошибки
"ErrorCode" - число, код результата выполнения метода:

    // 300   - не заполнен обязательный параметр "Date".

    // 410   - длина параметра "StartTime" не верна.    
    // 500   - не заполнен обязательный параметр "EndTime".

    // 800   - не найден врач с указанным ИД.
    // 900   - отказ записи на прием - время пациента уже занято другим или этим же пациентом
